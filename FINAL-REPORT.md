# FINAL REPORT

## Overview
This experiment was about seeing what happens when we run the exact same prompt many times using the same AI coding tool. We wanted to see how much the outputs changed, how good or bad they were, and whether one refinement turn actually helped that much.

## Setup
We used Claude Code with the same model every time. Each run was done in a clean session, and we did not hand-edit the generated code. For later rounds, we only did one refinement turn each time.

## What We Found in Step 1
There was a lot more variation than expected even though the prompt stayed the same. Some candidates were simple, some had a lot more features, and some worked better in the browser than others. A few had stronger AI joke themes, while others felt more like a normal slot machine and lost the point of the prompt.

## What Happened During Refinement
Refinement helped some candidates, but mostly the ones that were already strong. It usually improved visual effects, color palette, or polish more than deeper problems. If a candidate already had a weak base, refinement did not fully fix it.

## Final Candidate
The final winner was candidate-010. It stayed strong through all the rounds and ended up having the best overall UI, effects, and AI sarcasm theme. It also felt the most complete.

## Main Takeaway
The main thing we learned is that even with the same exact prompt, the outputs can still vary a lot. AI can give some decent prototypes quickly, but the quality is not very consistent. Refinement helps, but it helps the best candidates more than the weaker ones.

## Limitations
This was only one kind of task, one prompt, and one workflow. It does not prove that AI coding tools are always good or always bad. It just shows what happened in this specific experiment.

## Conclusion
Overall, the experiment showed that AI can make usable prototypes, but there is still a big spread in quality. The best strategy was not expecting every run to be good, but picking the strongest ones and refining those.
